## Usage

After installing, register the Cursor ACP provider inside your gsd2 configuration so GSD-2 loads it at startup.

### 1. Add the provider to your gsd2 config

In your gsd2 project, open (or create) `.gsd2/config.ts` (or the equivalent config entry point your gsd2 version uses) and call `registerCursorAcpProvider` before any call that resolves a model:

```typescript
import { registerCursorAcpProvider } from '@gsd/pi-ai-cursor-acp';

// Call once at startup, before gsd2 resolves any model
await registerCursorAcpProvider();
```

This registers all Cursor-available models under the `cursor-acp` provider namespace. If `cursor-agent` is not found on your PATH the call throws `CursorCliNotFoundError` (see Error Handling below).

### 2. Select a Cursor model in your gsd2 config

Once registered, any Cursor model that `cursor-agent --list-models` returns is available. Reference it by its discovered ID:

```
cursor-acp/<model-name>
```

For example, in a `.gsd2/config.yaml` (syntax depends on your gsd2 version):

```yaml
model: cursor-acp/claude-3-5-sonnet
```

### 3. Verify the integration

Run gsd2 normally. If the provider is correctly wired, the first invocation will spawn `cursor-agent` and establish an ACP session. Check the gsd2 logs for a `cursor-acp` provider line at startup.
```
