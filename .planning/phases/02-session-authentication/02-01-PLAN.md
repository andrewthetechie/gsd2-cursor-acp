    Exported types:
    ```typescript
    export type PermissionPolicy = 'auto-approve-all' | 'approve-reads-reject-writes' | 'interactive';
    export type RequestPermissionOutcome = { outcome: 'cancelled' } | { outcome: 'selected'; optionId: string };
    ```

    Class structure:
    ```typescript
    export class PermissionHandler {
      constructor(private readonly policy: PermissionPolicy) {}

      resolvePermission(
        toolKind: string | undefined,
        options: PermissionOption[],
      ): RequestPermissionOutcome { ... }

      private selectOption(
        options: PermissionOption[],
        preferredKind: PermissionOptionKind,
      ): RequestPermissionOutcome { ... }
    }
    ```

    Policy dispatch in resolvePermission:
    - `auto-approve-all`: Always call `this.selectOption(options, 'allow_once')` (per AUTH-02 default behavior).
    - `approve-reads-reject-writes`: Check if toolKind is in the read-safe set (`'read'`, `'search'`, `'think'`, `'fetch'`). If yes, select `allow_once`. If toolKind is in write-dangerous set (`'edit'`, `'delete'`, `'move'`, `'execute'`), select `reject_once`. For `'switch_mode'`, `'other'`, or `undefined`, fall back to first option in array.
    - `interactive`: Return `{ outcome: 'cancelled' }` immediately.

    selectOption implementation (per Pitfall 2 -- never hardcode optionId):
    - Find option in array where `option.kind === preferredKind`.
    - If found, return `{ outcome: 'selected', optionId: option.optionId }`.
    - If not found, fall back to first option: `{ outcome: 'selected', optionId: options[0].optionId }`.

    Import `PermissionOption` and `PermissionOptionKind` from `@agentclientprotocol/sdk`. If SDK does not export `PermissionOptionKind` as a type directly, define it locally as `type PermissionOptionKind = 'allow_once' | 'allow_always' | 'reject_once' | 'reject_always'`.

    Do NOT add logging, events, or cleanup methods -- keep this pure and simple. AcpSessionPool (Plan 02) handles the wiring.
