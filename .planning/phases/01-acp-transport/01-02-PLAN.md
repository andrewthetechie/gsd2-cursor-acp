```typescript
export class AcpTransport extends EventEmitter {
  // Public API
  async start(): Promise<void>          // Spawn child process, set up readline
  async sendRequest(method: string, params?: unknown): Promise<unknown>  // Send JSON-RPC request, return promise
  sendResponse(id: number | string, result: unknown): void  // Respond to server-initiated request
  sendNotification(method: string, params?: unknown): void  // Send notification (no response expected)
  async shutdown(): Promise<void>       // Graceful shutdown per D-04
  isRunning(): boolean                  // Check if process is alive

  // Events emitted
  // "notification" - JsonRpcNotification received
  // "request" - JsonRpcServerRequest received (needs sendResponse)
  // "error" - Fatal error (e.g., ProcessCrashError after 2 crashes in 30s)
  // "restarting" - Process crashed, auto-restarting
  // "started" - Process (re)started successfully
}
```

Implementation details:
- Constructor accepts `TransportOptions` (merged with DEFAULT_TRANSPORT_OPTIONS)
- Per D-05: default binary is "cursor-agent" with args ["acp"]
- Use `spawn(binaryPath, binaryArgs, { stdio: ["pipe", "pipe", "pipe"] })` -- pipe stderr too so we can capture it for debug logging
- Use `readline.createInterface({ input: process.stdout })` for line parsing
- `handleLine(line)`: try JSON.parse; on failure, emit "warning" and skip. Then route:
  1. Has `id` AND (`result` !== undefined OR `error` exists) -> response, resolve/reject pending
  2. Has `method` AND `id` -> server-initiated request, emit "request"
  3. Has `method` (no `id`) -> notification, emit "notification"
- `sendRequest`: assign `this.nextId++`, write JSON + "\n" to stdin, create Promise with timeout timer, store in `this.pending` Map
- On process "exit" event: if not shutting down, apply D-03 crash/restart logic
- On process "error" event (e.g., ENOENT if binary not found): emit "error" with TransportError
- `shutdown()`: set `this.shuttingDown = true`, reject all pending, send SIGTERM, race between "exit" event and 5s timeout, SIGKILL if needed
- Register host process cleanup: `process.on("exit", () => this.shutdownSync())` and `process.on("SIGTERM", ...)`, `process.on("SIGINT", ...)`
